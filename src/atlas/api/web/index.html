Why Atlas

Built on the principle that
wrong answers cost more than slow ones

Most RAG systems optimize for fluency. Atlas optimizes for trustworthiness — every response is grounded, traceable, and honest about its limits.

Faithfulness checking

After generating an answer, Atlas decomposes it into individual claims and verifies each one against the retrieved sources. Unsupported claims are flagged — not silently included.

Honest out-of-scope refusal

The query router classifies every question before retrieval. If the topic isn't covered by the indexed corpus, Atlas declines to answer rather than confabulate a plausible-sounding response.

Two-stage hybrid retrieval

BM25 catches exact keyword matches; dense embeddings catch semantic meaning. Reciprocal Rank Fusion merges the two lists, then a cross-encoder reranker picks the best 5 — precision over recall.

How it works

Every query runs the same
five-stage decision tree

Atlas doesn't call a single LLM and return whatever comes back. The pipeline routes, retrieves, grades, generates, and verifies — with each stage able to halt or retry before the next one runs.

Query router
Classifies the question as simple, complex, or out-of-scope. Out-of-scope queries exit here — no retrieval, no generation.
Decomposer
Complex multi-hop questions are broken into up to 4 sub-questions, each retrieved independently, then merged before grading.
Hybrid retrieval + reranker
Dense (Qdrant ANN) + sparse (BM25) search runs in parallel. RRF fuses the ranked lists; a cross-encoder reranks the top 20, returns the best 5.
Retrieval grader
Scores whether the retrieved context is sufficient to answer the query. If not, reformulates and retries up to 2 times before passing to the generator.
Answer generator
Writes the answer with inline citations [1], [2] — each linked to the source chunk. The prompt explicitly forbids fabricating facts not in the context.
Faithfulness checker
Decomposes the answer into individual claims. Verifies each claim against the retrieved chunks. Flags any that aren't grounded.

Reciprocal Rank Fusion

BM25 and cosine similarity scores live on incompatible scales — you can't simply average them. RRF uses ranks instead: score = Σ 1/(k+rank) where k=60. Any retriever can contribute without calibration.

Two-level cache

An in-memory LRU (256 entries) sits in front of Redis (1h TTL). Cache keys are the lower-cased query hash. Repeated questions are answered in <1ms — zero LLM cost.

Idempotent indexing

Every document chunk is fingerprinted with xxhash. Re-running the ingestion pipeline skips unchanged files. Safe to run on a schedule without duplicating your index.

Per-request tracing

Each request gets a UUID bound to structlog context. The same ID appears in every log line from that request across all pipeline stages — and surfaces in response headers as X-Request-Id.

Get started

From zero to cited answers
in five commands

Requires Docker (for Qdrant + Redis), Python 3.11, and an OpenAI API key. The full pipeline — ingestion, retrieval, faithfulness checking — runs locally.

01

Add your OpenAI key

Copy .env.example to .env and fill in OPENAI_API_KEY. Everything else has sane defaults for local development.

02

Start the infrastructure

Qdrant (vector store) and Redis (query cache) run in Docker. The API server runs on your machine.

03

Fetch and index a corpus

The bundled script downloads the FastAPI documentation (MIT-licensed, ~120 files). Index it with the ingestion pipeline.

04

Start the API server

Swagger UI at /docs, the console at /app, Prometheus metrics at /metrics.

05

Ask a question

Use the console, call POST /query directly, or run scripts/seed_demo.py to see 5 live queries with full provenance printed to the terminal.

bash 
# 1. env
cp .env.example .env
# set OPENAI_API_KEY=sk-…

# 2. infrastructure
docker-compose up qdrant redis -d

# 3. install + corpus
uv pip install -e ".[dev]"
python scripts/fetch_corpus.py
python scripts/ingest.py data/corpus/fastapi/

# 4. start API
uvicorn atlas.api.asgi:app --reload --port 8010
# → http://localhost:8010/app

# 5. run demo
python scripts/seed_demo.py

See the pipeline in action

The console shows the full trace — routing, retrieval, grading, generation, faithfulness verdict — for every query you ask.
